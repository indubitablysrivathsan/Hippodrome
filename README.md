# Horse Racing Signal Processing Research

**Signal Processing Analysis of Indian Horse Racing Betting Markets**

This repository applies information-theoretic and digital signal processing (DSP) methods to analyse Indian horse racing betting markets. Each race's odds distribution is modelled as a probabilistic signal from which Shannon entropy is computed, producing a discrete-time entropy signal spanning ~7,000 races across multiple Indian venues (2010–present). Standard DSP techniques — autocorrelation, FFT, power spectral density, discrete and continuous wavelet transforms, STFT, bandpass filtering, denoising, and Lempel-Ziv compressibility — are applied to study temporal structure, periodicity, regime changes, and market efficiency.

## Setup

```bash
# Clone the repository
git clone https://github.com/indubitablysrivathsan/Hippodrome
cd Hippodrome

# Place raw data files in data/raw/
# (meetings.csv, races.csv, runners.csv, exotics.csv)
```

---

## Running the Pipeline

The pipeline transforms raw CSV data into processed analysis-ready files.

```bash
python pipeline/01_parse_odds.py        # → data/processed/entries_market.csv
python pipeline/02_build_race_features.py  # → data/processed/race_features.csv
python pipeline/03_build_signal.py       # → data/processed/entropy_signal.csv
python pipeline/04_build_calibration.py  # → data/processed/calibration_bins.csv
```

Each script prints a validation summary showing row counts, statistics, and data quality checks.

---

## Running the Notebooks

```bash
cd Hippodrome
jupyter notebook
```

Open any notebook from the `notebooks/` directory. Notebooks must be run **after** the pipeline has completed, as they read from `data/processed/`.

---

## Running Analysis Scripts

Each analysis script produces publication-quality figures saved as both PNG and PDF:

```bash
python analysis/entropy_distribution.py
python analysis/market_efficiency.py
python analysis/calibration_curve.py
python analysis/lz_compressibility.py
python analysis/regime_detection.py
```

All figures are saved to `outputs/figures/`.

## Processed File Descriptions

| File | Rows | Description |
|------|------|-------------|
| `entries_market.csv` | ~68,000 | One row per runner per race. Includes decimal odds, implied probabilities, probability rank, and upset magnitude. |
| `race_features.csv` | ~7,100 | One row per race. Includes Shannon entropy, normalised entropy, overround, winner statistics, and track conditions. |
| `entropy_signal.csv` | ~7,100 | Chronologically ordered entropy time series — the primary DSP input signal. |
| `calibration_bins.csv` | ~100 | Aggregated calibration data (overall + per venue) for calibration curve analysis. |

## Data Sources

Indian horse racing public data (2010–present). Raw data includes race meetings, race cards, runner entries with fractional odds, and exotic bet pools from major Indian racing venues: Mumbai, Bangalore, Chennai, Hyderabad, Kolkata, Delhi, Mysore, and Pune.
